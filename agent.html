Modules Clés
🔄
AGENT LOOP
core/agent_loop.py
  • Boucle while True avec état machine
  • Déclencheurs : demande user, watcher, timer, erreur
  • Gestion des états : IDLE, THINKING, ACTING, RECOVERING
  • Signal propre pour arrêt (SIGTERM / Ctrl+C)
👁️
WATCHER
core/watcher.py
  • Surveille neron/ avec watchdog
  • Détecte modifications, ajouts, suppressions
  • Debounce pour éviter les faux positifs
  • Pousse les événements dans la queue agent
🧪
SANDBOX
core/sandbox.py
  • Exécute le code généré dans un sous-process isolé
  • Timeout configurable (défaut 10s)
  • Capture stdout / stderr / code retour
  • Jamais d'écriture sans validation sandbox
💾
BACKUP
core/backup.py
  • Snapshot automatique avant toute auto-modification
  • Horodatage + hash du fichier original
  • Rollback en une commande
  • Rotation des backups (garde N derniers)
🧠
MEMORY
core/memory.py
  • Persistance SQLite (data/memory.db)
  • Tables : actions, résultats, erreurs, améliorations
  • Requêtes : "qu'est-ce qui a échoué récemment ?"
  • Contexte injecté dans chaque prompt LLM
🔧
SELF TOOL
tools/self_tool.py
  • Lit son propre code source
  • Soumet à Nemotron pour analyse de qualité
  • Propose des patches ciblés (pas de réécriture totale)
  • Toujours via backup + sandbox avant application
⚡
HYBRID LLM
llm/hybrid.py
  • SDK Ollama officiel (plus de subprocess)
  • Phi3 → génération de code
  • Nemotron → orchestration + validation
  • Retry automatique sur timeout/erreur
📋
LOGGER
core/logger.py
  • Log structuré JSON dans data/logs/
  • Chaque décision de l'agent tracée
  • Rotation quotidienne des fichiers
  • Niveau configurable (DEBUG / INFO / WARN)

⚡ Garde-fous Critiques

Backup avant toute écriture sur soi
Sandbox obligatoire avant apply
Path traversal bloqué (pathlib)
Timeout sur tous les LLM calls
Rollback automatique si erreur post-apply
Mode dry-run disponible
Log immuable de chaque action
Arrêt propre sur signal système
